## Commands

Package manager is **bun**. Node `^20.19.0 || >=22.12.0`.

- `bun dev` — Vite dev server with hot reload
- `bun run build` — parallel `type-check` + `build-only` (production build)
- `bun run type-check` — `vue-tsc --build` (the only typecheck step; `tsc` is replaced because of `.vue` files)
- `bun lint` — runs `oxlint --fix` then `eslint --fix --cache` in sequence (`run-s lint:*`)
- `bun run lint:oxlint` / `bun run lint:eslint` — run individually
- `bun run format` — `oxfmt src/`
- `bun run preview` — preview the production build

Card-data scripts (run with bun directly, not npm scripts):

- `bun scripts/fetchCards.ts <SET_CODE>` — pulls a Magic set from Scryfall, writes `src/sets/<SET>-card-base.json`
- `bun scripts/reduceFields.ts <INPUT> <OUTPUT>` — slims a card JSON to `name`, `colors`, `rarity`, `image_uris` (fields are hardcoded in the script, not CLI args despite the usage message)

No test runner is configured.

## Architecture

Single-page Vue 3 (beta) + Vite + TypeScript app. The app is a one-shot submission form for a Magic: The Gathering pick contest — users choose one card per color/guild slot from a fixed set, plus a Mystical Archive tiebreaker, and the submission is written to Firestore keyed by display name.

**Entry & UI stack**

- `src/main.ts` mounts `App.vue`, registers `@nuxt/ui` plugin. Tailwind v4 is wired through Nuxt UI's Vite plugin.
- `App.vue` renders `<UApp>` → `ContestExplanation` + `SubmissionForm`. No router.
- Nuxt UI auto-imports components and composables; generated declarations live in `auto-imports.d.ts` and `components.d.ts` (gitignored, regenerated by Vite).
- Path alias `@/` → `src/` (configured in `vite.config.ts` and `tsconfig.app.json`).

**Form flow** (`src/components/`)

- `useCards.ts` loads two static JSON sets (`SOS-commons-uncommons.json`, `SOA-cards.json`) and exposes a `mapOfCards` keyed by slot (`whiteCommons`, `blueCommons`, …, `wbCards` Silverquill, `wrCards` Lorehold, `urCards` Prismari, `ugCards` Quandrix, `gbCards` Witherbloom). Slots are filtered by exact color identity via `isCardExactlyColors`. Mystical Archive cards are returned separately for the tiebreaker.
- `SubmissionForm.vue` owns form state, valibot schema (display name validation: no slashes, no double underscores, no spaces, non-empty), and the `SuccessModal`. Each `CardChoice` row renders a collapsible list of cards; clicking opens a modal preview before confirming the pick.
- `useSubmission.ts` validates that every slot + tiebreaker is selected, then calls `handleSubmission`.
- `handleSubmission.ts` runs a Firestore transaction against the `submissions-sos` collection, using the display name as the doc id. Throws `DisplayNameInUse` on conflict; returns `'success' | 'error-display-name-in-use' | 'error-unknown'`.
- `composables/useToasts.ts` wraps Nuxt UI toasts for error display.

**Firebase** — `src/firebase.ts` initializes the app with hardcoded public web config (Firestore only). The collection name `submissions-sos` is hardcoded in `handleSubmission.ts`; if running a new contest, change it there.

**Card data** — `src/sets/*.json` is the slimmed output of the `fetchCards.ts` → `reduceFields.ts` pipeline. The `Card` type (`name`, `colors`, `rarity`, `image_uris.{normal, art_crop}`) is defined in `CardChoice.vue` and re-imported elsewhere.

**`packages/analysis/`** — currently only contains `PLAN.md`, a design doc for a future results-visualization page (static-JSON-driven, not yet implemented).

## Linting

Two linters run in series. **oxlint runs first** and is the fast pass; eslint runs second with `eslint-plugin-oxlint` to avoid duplicate rules. Configs: `.oxlintrc.json`, `.oxfmtrc.json`, `eslint.config.ts`. When fixing lint issues, run `bun lint` to apply both.

## Vue version pinning

`package.json` pins `vue: beta` and **overrides every `@vue/*` package to `beta`**. Don't bump individual `@vue/*` deps — keep them on `beta` together, or replace the whole block.
